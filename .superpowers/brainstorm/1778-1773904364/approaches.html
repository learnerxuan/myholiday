3 Approaches — How AI ↔ Itinerary Panel Sync Works

The chat + map + panel are connected. The key question is: how does a "confirmation" in chat update the panel and map?

A

Structured AI Response ⭐ Recommended

Every AI reply returns two things simultaneously: a prose message for the chat bubble, and a structured JSON diff for the panel/map. Frontend parses the JSON to update itinerary state and drop map pins — automatically, no extra clicks.

// AI response shape
{
  message: "✓ Added Kyoto Garden Inn! Next, Day 1 activities?",
  itinerary_update: {
    day: 1, type: "hotel",
    name: "Kyoto Garden Inn",
    lat: 35.0116, lng: 135.7681,
    status: "confirmed"
  }
}

Pros

  • Panel updates instantly as AI responds
  • Map pin drops the moment something is confirmed
  • No extra user clicks needed

Cons

  • Need OpenAI structured output / response format
  • Slightly more complex API route logic
B

Explicit Confirm Button in Options Tab

AI presents hotel/restaurant options as cards in the Options tab. User clicks "Select" on a card to officially confirm it. That action updates the itinerary panel and drops the map pin. Chat is just conversation — panel is controlled by explicit clicks.

Pros

  • User has full control — nothing added without their click
  • Simpler AI response (just prose, no structured JSON)
  • Clearer UX — user knows exactly what they confirmed

Cons

  • User must switch to Options tab to confirm
  • Less "AI magic" feel — more manual
  • Conversation and panel feel disconnected
C

Supabase Realtime Sync via update_itinerary Tool

When user confirms something in chat, the LLM calls an update_itinerary tool. That tool writes to Supabase. The right panel subscribes to Supabase Realtime and updates live — even if you opened the page on another device.

Pros

  • Itinerary persisted in DB as it's built (no data loss)
  • Cross-device: start on phone, continue on laptop
  • Clean separation between chat and itinerary state

Cons

  • Most complex to implement
  • Tool-call round trip adds latency
  • Realtime subscription setup needed

Map: Route Drawing by Day (B+C)

Tech: Leaflet.js + OpenStreetMap (free, no API key) + Leaflet Routing Machine plugin (draws routes using OSRM, also free)

Day filter: Buttons above the map — Day 1 / Day 2 / Day 3 / All. Switching days redraws only that day's pins and route.

Pins: 🏨 Hotel · 🍽 Restaurant · 🎯 Attraction · 🚌 Transport stop — different icons per type

Route line: Connects places in the order they appear in the itinerary for that day. Walking or driving route.
