How the AI Chatbot Actually Works

Your intuition is mostly right — here's the precise picture

Your Understanding ✓ (Correct)

The AI is a decision-maker and orchestrator — it receives user input, decides what to do, calls tools/scripts, and synthesises the output. It does not browse the internet on its own.

The Full Flow

User
"I want halal food, 3-star hotel, 5 days in Kyoto"
↓ POST /api/chat
Next.js API Route (Server-side)
→ Sends message + conversation history to LLM
→ LLM decides which tools to call:
• get_weather("Kyoto", "May") → queries your DB (avg_temp_monthly)
• get_hotels("Kyoto", "3-star", budget) → queries seeded hotel data
• get_halal_restaurants("Kyoto") → queries seeded restaurant data
→ LLM receives tool results
→ LLM synthesises a day-by-day itinerary
↓ response
User sees
"Here's your 5-day Kyoto plan: Day 1 – Check into Kyoto Garden Inn (3★, RM 280/night)..."

What the LLM Can and Cannot Do

✓ CAN DO
  • Call your backend tools/scripts
  • Use data from your Supabase DB
  • Generate text from training knowledge
  • Remember the conversation history
  • Decide which tool to call when
✗ CANNOT DO
  • Browse the internet live
  • Access real-time hotel prices (unless you call a 3rd-party API)
  • Access real-time flight data
  • Know events happening after its training cutoff

What "backend scripts" means in practice

You seed your Supabase DB with destination data. The LLM calls your backend functions, which query the DB and return structured results. The LLM then writes the itinerary using those results. No live internet needed.

Your destinations table already has avg_temp_monthly (JSONB) — so weather is already in the DB. The AI just queries it.
