Script vs MCP — Which one is search_hotels?

Short answer: a backend script via Function Calling. MCP is a different thing.

A

Function Calling (what you want)

You write search_hotels() as a plain JS function inside your Next.js backend. You define it as a "tool" for the LLM. The LLM decides when to call it, your server executes it, result goes back to LLM.

✓ Fits your Next.js app perfectly  ✓ Standard pattern  ✓ Works with OpenAI & Gemini

B

MCP (not what you want here)

MCP (Model Context Protocol) is Anthropic's protocol for connecting Claude (the model in Claude.ai / Claude Code) to external services. It's for tool builders, not for embedding AI in your own web app.

✗ Doesn't apply to a Next.js web app  ✗ For Claude desktop integrations, not your chatbot

How Function Calling Works in Your App

// Step 1: You define your tools (once, in /api/chat/route.js)
const tools = [
{ name: "search_hotels", parameters: { city, budget, guests } },
{ name: "get_weather", parameters: { city, month } },
{ name: "search_restaurants", parameters: { city, cuisine, priceRange } }
]
// Step 2: Send user message + tools to LLM
const response = await openai.chat({ messages, tools })
// Step 3: LLM replies → "call search_hotels(Kyoto, budget, 2)"
if (response.tool_call) {
const result = await search_hotels(response.tool_call.args)
// your function runs here — calls Google Places API, returns list
}
// Step 4: Send result back to LLM → LLM writes final reply with cards
await openai.chat({ messages, tool_result: result })

TL;DR

search_hotels = a plain JavaScript function in lib/ai/tools/search_hotels.js that calls Google Places API. The LLM calls it via OpenAI Function Calling (or Gemini's equivalent). MCP is a completely separate thing used for Claude.ai integrations — not relevant to your web app.
